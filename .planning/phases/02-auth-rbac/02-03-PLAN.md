## Phase Goal

**As a** user who forgot my password or needs email verification,
**I want to** receive a verification/reset email and complete the flow,
**so that** I can sign in securely; and as an admin, **I want to** ban users and revoke sessions server-side,
**so that** compromised or former accounts are immediately neutralized.

This plan delivers the email flows slice (AUTH-06/07) and the ban + session primitives slice (D-16/D-17). It closes out Phase 2 by replacing the Plan 02-01 email-hook stubs with the real Resend-backed helper and adding the ban/revoke test coverage.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Better Auth hook → lib/email → Resend API | Server-only; the Resend key crosses here and must never reach the client |
| admin → banUser/revokeUserSessions | Admin-only mutations; each re-checks requireCan({user:['ban'/'revoke-session']}) |
| Email link click → /api/auth/* | Verification/reset tokens cross here; Better Auth validates token + expiry + single-use |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-06 | Information disclosure | Resend API key leak to client | high | mitigate | lib/email is server-only (no "use client"); key in env only; never imported by a client component (ASVS V8) |
| T-02-04 | Information disclosure | email enumeration via sign-up/sign-in errors | medium | mitigate | requireEmailVerification:true + customSyntheticUser (returns generic synthetic user with admin-plugin fields — Code Examples lines 860-879) |
| T-02-08 | Information disclosure | timing attack on email-send hooks | medium | mitigate | All hooks use `void sendEmail(...)` (fire-and-forget); lib/email returns silently on error — R8 |
| T-02-10 | Spoofing/Tampering | forged/intercepted verification or reset token | high | mitigate | Better Auth generates + validates single-use tokens with expiry; the verification/reset tables enforce it (Don't Hand-Roll table) |
| T-02-08b | Elevation | banned user retains active session | high | mitigate | admin.banUser revokes all sessions immediately per docs (Stale-session-after-ban threat in Security Domain) |
| T-02-SC | Tampering | npm installs | high | mitigate | No new npm packages this plan. resend was verified in 02-01 (SUS flag is a false positive — canonical locked dep). |
</threat_model>
