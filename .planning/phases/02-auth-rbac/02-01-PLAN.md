## Phase Goal

**As a** member of the editorial team (admin/editor/author),
**I want to** sign into a dashboard gated by role-based permissions with server-side enforcement,
**so that** the review workflow is genuinely enforced (not decorative) when posts ship in Phase 3.

This plan delivers the foundational slice: the test runner, the auth schema migration, the Better Auth instance with RBAC config, the permission helpers, the proxy UX gate, and the status-transition helpers. Every later Phase 2 plan and all of Phase 3 depends on these artifacts.
