## Decisions Made
- **Drizzle 0.45.2 pinned (not 1.x):** Better Auth's peer dependency (`^0.45.2`) locks drizzle-orm here. Drizzle 1.0 is RC but Better Auth doesn't accept it yet. Do NOT upgrade until Better Auth bumps its peer.
- **Deferred FKs (D-07):** `posts.authorId` and `posts.categoryId` are plain `integer()` columns with NO `.references()` call. The FK constraints are added in Phase 2 when the users table lands. This matches RESEARCH.md Pattern 2 verbatim and avoids any forward-reference concern.
- **Pages own SEO (D-06):** The `pages` table carries `metaTitle`, `metaDescription`, `canonical` directly — not a polymorphic/shared SEO table. `post_seo` remains a separate one-to-one table for posts.
- **Soft-delete split (D-08):** `deletedAt: timestamp("deleted_at")` on posts, pages, media, categories, tags (content tables). Omitted on settings, postTags, postSeo (join/utility tables use hard-delete).
- **settings PK is varchar key:** The settings table uses `varchar("key", { length: 255 }).primaryKey()` — a key-value config store, not a serial-id table.
