<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build-time config → runtime | next.config.ts values (cacheComponents, remotePatterns) cross from config into the runtime image optimizer — a misconfigured remotePattern could allow arbitrary external image fetching |
| Env vars → client bundle | NEXT_PUBLIC_CDN_URL is the only NEXT_PUBLIC_ var; it is safe to expose (a public CDN hostname). All other env vars (DATABASE_URL, S3 secrets) must never be prefixed with NEXT_PUBLIC_. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | next/image remotePatterns | medium | mitigate | Whitelist exactly two hosts (cdn.anydiscussion.com + localhost:9000 MinIO); reject wildcard patterns. Prevents the image optimizer from being abused as an SSRF proxy. |
| T-01-02 | Information Disclosure | src/app/error.tsx | medium | mitigate | Error boundary renders a generic message to the user; full error details logged server-side only via lib/log; never display error.stack or raw error.message containing internal paths to the client. |
| T-01-03 | Tampering | ESLint isolation rule | high | mitigate | Literal glob patterns (not regex) for no-restricted-imports; the rule is a "error" not "warn"; verified by a planted cross-group import in Plan 03's pnpm verify. Prevents TailAdmin/editor JS leaking into the public bundle (foundation of Phase 7 PERF-02). |
| T-01-04 | Information Disclosure | env var leakage | high | mitigate | Only NEXT_PUBLIC_CDN_URL is client-exposed (safe). All secrets (DATABASE_URL, S3_ACCESS_KEY_ID, S3_SECRET_ACCESS_KEY) live in gitignored .env.local. .gitignore line 30 already covers .env*.local. |
| T-01-SC | Tampering | pnpm package installs | high | mitigate | All packages (drizzle-orm, drizzle-kit, pg, @aws-sdk/client-s3, sharp, @types/pg) are canonical, verified in RESEARCH.md Package Legitimacy Audit (all OK/Approved). No [ASSUMED] or [SUS] packages requiring a blocking human checkpoint. sharp postinstall gated by pnpm approve-builds allowlist. |
</threat_model>
