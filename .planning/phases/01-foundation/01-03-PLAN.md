<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App → S3-compatible storage | The lib/r2 helper writes objects to MinIO/R2 — the boundary is the s3Client singleton; keys are server-generated (baseKey parameter), not from untrusted user input in Phase 1 |
| Dev machine → Docker services | Docker Compose exposes Postgres (5432/5433) and MinIO (9000/9001) on localhost — acceptable for local dev, must not be exposed externally |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-09 | Tampering | Path traversal in R2 keys | medium | mitigate | The baseKey parameter in uploadImageVariants is server-generated (e.g., "posts/2026/07/my-image"). No user-supplied path component in Phase 1. Phase 3 must sanitize user-uploaded filenames before they reach baseKey. |
| T-01-10 | Denial of Service | Image bomb (decompression attack) via sharp | medium | mitigate | sharp has built-in pixel/byte limits. For Phase 1's minimal helper, the input is a developer test buffer. Phase 3 (MEDIA-01) must add input validation (max file size, max dimensions, magic-byte format check) before calling sharp. |
| T-01-11 | Information Disclosure | S3 credentials in env | high | mitigate | S3_ACCESS_KEY_ID and S3_SECRET_ACCESS_KEY live only in gitignored .env.local for real secrets. .env.example ships MinIO defaults (minioadmin/minioadmin) which are safe for local dev only. Never prefixed NEXT_PUBLIC_. Coolify injects real R2 keys at deploy. |
| T-01-12 | Tampering | Docker ports exposed externally | low | accept | Docker Compose binds to localhost:5432/5433/9000/9001 by default on the dev machine. Not deployed in Phase 1. Acceptable risk for local dev; Phase 7 deployment hardening covers production network config. |
| T-01-SC | Tampering | No new package installs in this plan | low | accept | All packages (drizzle-orm, pg, @aws-sdk/client-s3, sharp) were installed and audited in Plan 01. No additional installs here. |
</threat_model>
