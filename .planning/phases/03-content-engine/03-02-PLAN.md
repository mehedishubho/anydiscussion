## Phase Goal

**As a** reader of the public blog,
**I want to** never be served malicious scripts or disallowed iframes from author-pasted content,
**so that** the post body and taxonomy are safe end-to-end regardless of what an author pastes into the editor.

This is **Slice B** of the Phase-3 MVP vertical-slice plan: shared sanitization + taxonomy. The single DOMPurify config is the linchpin of Pitfall #2 (double-sanitization) — Slice A's posts.ts stores ProseMirror JSON safely (no HTML stored in posts.body), but the embed-paste path (D-02 raw HTML in editor content) and the SSR render pipeline (generateHTML → dangerouslySetInnerHTML) both need the shared sanitize gate. The taxonomy pickers wire D-23 (required category, tags capped at 8) into the editor UI Slice A shipped.

## Artifacts this phase produces

**Exports / symbols:**
- `sanitizeBeforeStore(html)` — DOMPurify.sanitize with the shared config (site #1 — called in posts.ts before db.insert for any HTML-capable field, including the raw-HTML embed nodes inside the ProseMirror JSON when pre-serialized)
- `sanitizeBeforeRender(html)` — DOMPurify.sanitize with the SAME shared config (site #2 — called in post-render.ts before dangerouslySetInnerHTML)
- `EMBED_DOMAIN_ALLOWLIST` — the per-provider iframe domain allowlist constant (YouTube/X/Instagram/Vimeo/Soundcloud and their subdomains)
- `renderPostBody(postBodyJson)` — the complete SSR pipeline: generateHTML → sanitizeBeforeRender
- CategoryPicker / TagPicker / TaxonomyPicker components for the post editor

**Files created:** 6 new files (1 sanitize module, 1 post-render module, 3 picker components, 1 sanitize test). Files modified: actions/posts.ts (wire sanitizeBeforeStore on the embed HTML extraction path), actions/categories.ts + tags.ts (no schema change — confirm softDelete + slug validation already shipped in Slice A), posts/new + edit pages (wire TaxonomyPicker), posts.test.ts + taxonomy.test.ts (extend with sanitize + picker assertions).

**No schema migration in this slice** — Slice A shipped the migration; this slice composes the existing tables.
