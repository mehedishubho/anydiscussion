## Phase Goal

**As an** editor uploading an image to the dashboard,
**I want to** upload through a configurable storage provider that produces optimized variants and serves them via next/image,
**so that** the active provider is selectable from settings (local default, R2 available) without changing the upload code path.

This is **Slice C** of the Phase-3 MVP vertical-slice plan: media + storage abstraction. It consumes the `media` table columns Slice A migrated (`providerKey`, `provider`, `uploadedBy` text FK) and the existing Phase-1 `lib/r2` `uploadImageVariants` pipeline (D-09 — wrapped unchanged by the R2 provider). The Cloudinary + push-CDN providers are explicitly Phase 4 (DASH-09) — this slice ships local + R2 only.

## Artifacts this phase produces

**Exports / symbols:**
- `StorageProvider` — the interface (`upload`, `getPublicUrl`, `delete`) (`src/lib/storage/types.ts`)
- `UploadedVariant` — reused/extended from `src/lib/r2` (key, width, height, format, sizeBytes)
- `getActiveProvider()` — the registry entry point reading `settings.storage.active_provider` (`src/lib/storage/registry.ts`)
- `registerStorageProvider(name, provider)` — registration helper for Phase-4 Cloudinary/push-CDN extensibility
